# Mammalian Phenotype Ontology (MP)

This document describes how the [Mammalian Phenotype Ontology][obo-mp] is integrated
into Babel and what it contributes to the disease/phenotype compendium. Quantitative
counts (rows in, rows filtered, rows written) are not duplicated here — they live in the
concord-adjacent metadata files generated by the pipeline (see [Where to look at
run-time](#where-to-look-at-run-time)). This file should be kept in sync with the
snakefile that drives the integration: PRs that change source URLs or filters must
update this README in the same change.

## Overview

MP was added to satisfy a [request from the CAM-KP team][issue-240] to support querying
mouse-model phenotypic data through the Translator platform. The integration is
implemented by [PR #300][pr-300]. All MP terms are typed as `biolink:PhenotypicFeature`
and join the existing disease/phenotype compendium alongside HP, MONDO, DOID, EFO,
NCIT, MESH, UMLS, OMIM, ORPHANET, MEDDRA, SNOMEDCT, and ICD0/9/10.

The preferred-prefix order in the disease/phenotype builder is `[MONDO, HP, MP]`; MP is
never the leader of a clique that contains a MONDO or HP identifier. MP-only cliques
(those with no other vocabulary's identifier) keep the MP CURIE as their leader, which
means MP IDs only become canonical when no overlapping mapping exists.

| License | CC-BY 4.0 |
| Root term used by Babel | `MP:0000001` (the root Mammalian Phenotype concept) |

MP standard terms annotate mammalian phenotype data and are used by MGI (mouse), RGD
(rat), and IMPC (the International Mouse Phenotyping Consortium).

## Identifiers incorporated

Babel does not run a dedicated downloader for MP. Instead, identifiers are enumerated
through UberGraph by listing all subclasses of `MP:0000001`:

```text
write_mp_ids() (src/createcompendia/diseasephenotype.py)
  -> obo.write_obo_ids()             (src/datahandlers/obo.py)
    -> UberGraph.get_subclasses_of() (src/ubergraph.py)
```

Labels and synonyms come from the same UberGraph queries used for HP, MONDO, etc.,
driven by the `generate_dirs_for_labels_and_synonyms_prefixes` list in `config.yaml`.

## Cross-references downloaded

MP cross-references reach the disease/phenotype compendium through two pathways:

### UberGraph

`src/ubergraph.py` declares the `MP:` prefix in four SPARQL templates:
`get_subclasses_of`, `get_subclasses_and_smiles`, `get_subclasses_and_xrefs`, and
`get_subclasses_and_exacts`. Any MP↔X edge that already exists in the UberGraph
reasoner graph (xref / exactMatch / equivalentClass) will surface through the existing
`build_disease_obo_relationships` machinery without any MP-specific code.

### SSSOM mappings (Mouse-Human Ontology Mapping Initiative)

The [Mouse-Human Ontology Mapping Initiative][mh-init] curates HP↔MP equivalence
mappings as [SSSOM][sssom-spec] tables. Babel consumes seven of them via the
`get_hp_mp_concord` rule in `src/snakefiles/diseasephenotype.snakefile`. The function
that processes them is `build_hp_mp_concords` in
`src/createcompendia/diseasephenotype.py`.

| File | Source / scope |
|------|----------------|
| [`mp_hp_eye_impc.sssom.tsv`][eye-master] ([permalink][eye-perm]) | IMPC Mouse Morphology Mappings: Eye Morphology Test |
| [`mp_hp_hwt_impc.sssom.tsv`][hwt-master] ([permalink][hwt-perm]) | IMPC Mouse Morphology Mappings: Heart Weight Test |
| [`mp_hp_mgi_all.sssom.tsv`][mgi-master] ([permalink][mgi-perm]) | All MP↔HPO mappings generated by MGI (consolidated from KidsFirst, scoliosis, Alzheimer, cleft palate, Down Syndrome, CHARGE Syndrome, Cornelia De Lange Syndrome, COVID, etc.) |
| [`mp_hp_owt_impc.sssom.tsv`][owt-master] ([permalink][owt-perm]) | IMPC Mouse Morphology Mappings: Organ Weight Test |
| [`mp_hp_pat_impc.sssom.tsv`][pat-master] ([permalink][pat-perm]) | IMPC Mouse Morphology Mappings: Gross Pathology & Tissue Collection Test |
| [`mp_hp_pistoia.sssom.tsv`][pistoia-master] ([permalink][pistoia-perm]) | Pistoia Alliance Ontology Mappings: mp_hp (expert curation) |
| [`mp_hp_xry_impc.sssom.tsv`][xry-master] ([permalink][xry-perm]) | IMPC Mouse Morphology Mappings: X-ray Test |

The `master`-branch URLs are what the pipeline downloads. The permalinks point at the
SHA `3f3eeb05af9b600976b97185c5f5d8211a85df2c` of the `mh_mapping_initiative` repository,
which is the state when this document was first written (2026-05-01). The permalinks
exist so future readers can diff against today's master.

#### Filters applied to each SSSOM file

Implemented in `build_hp_mp_concords` (`src/createcompendia/diseasephenotype.py`):

1. **Confidence threshold** — drop rows where `confidence <= 0.8`. Rows with no
   `confidence` column are kept (this is how IMPC's files are treated, since they do
   not carry confidence values).
2. **`sssom:NoTermFound`** — drop rows where either `subject_id` or `object_id` equals
   `sssom:NoTermFound`.
3. **Predicate allowlist** — keep only rows whose `predicate_id` is one of
   `skos:exactMatch`, `skos:closeMatch`, or `skos:relatedMatch`.

Per-input row totals before and after each filter, and a per-predicate breakdown of
both kept and dropped rows, are written to `metadata-HP_MP.yaml` (see below).

## Pipeline wiring

| File | What it contributes |
|------|---------------------|
| `config.yaml` | Adds `MP` to `ubergraph_ontologies`, `disease_labelsandsynonyms`, `disease_ids`, and `generate_dirs_for_labels_and_synonyms_prefixes`. Adds `HP_MP` to `disease_concords`. |
| `src/prefixes.py` | Defines the `MP` prefix constant. |
| `src/ubergraph.py` | Declares the `MP:` SPARQL prefix in four query templates. |
| `src/createcompendia/diseasephenotype.py` | `write_mp_ids()` enumerates MP descendants through UberGraph. `build_hp_mp_concords()` filters the seven SSSOM tables and writes both the concord file and its metadata. The clique builder (`build_compendium`, `create_typed_sets`) uses preferred-prefix order `[MONDO, HP, MP]`. |
| `src/snakefiles/diseasephenotype.snakefile` | `disease_mp_ids` rule runs `write_mp_ids()`. `get_hp_mp_concord` rule runs `build_hp_mp_concords()`. |
| `pyproject.toml` | Adds `sssom` for parsing SSSOM TSV files. |

## Where to look at run-time

| Artefact | Path |
|----------|------|
| Concord file | `babel_outputs/intermediate/disease/concords/HP_MP` |
| Per-source + combined concord metadata | `babel_outputs/intermediate/disease/concords/metadata-HP_MP.yaml` |
| Final compendium | `babel_outputs/compendia/PhenotypicFeature.txt` (and `Disease.txt`) |
| Final compendium metadata | `babel_outputs/metadata/PhenotypicFeature.txt.yaml` |
| Per-rule benchmarks | `babel_outputs/benchmarks/disease_mp_ids.tsv`, `babel_outputs/benchmarks/get_hp_mp_concord.tsv` |

`metadata-HP_MP.yaml` records, for each of the seven inputs, the total input rows, the
confidence threshold used (or `null` if the column is absent), and how many rows were
dropped at each filter stage with per-predicate breakdowns. It also records the
top-level concord-walk counts (line count, distinct CURIEs, per-predicate, per-prefix
pair). That YAML is the authoritative source for "how many of what" — this README is
not.
